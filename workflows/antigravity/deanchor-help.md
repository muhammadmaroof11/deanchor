## Intensity Levels

Set the intensity via `/deanchor [lite|full|ultra]` (default: `full`):

*   **Lite:** Decouples data and identifies banned paradigms. Drafts **two** alternative concepts and pauses for user feedback before writing code.
*   **Full (Default):** The Deanchor Protocol strictly enforced. Decouples, bans, conceptualizes a single premium path, and writes the clean-slate code immediately.
*   **Ultra:** Radical redesign. Re-evaluates the core problem definition. Banishes all current tech stacks and frameworks. Proposes a minimal, alternative paradigm.

## Protocol Steps (The Core Reasoner)

Every redesign follows these four pillars:
1. **Decouple:** Extract raw state/data/copy, removing all layout or implementation structures.
2. **Ban:** List existing paradigms and label them `BANNED_PARADIGM`.
3. **Conceptualize:** Draft a blank-slate design that solves the decoupled requirements while avoiding the banned list.
4. **Execute:** Implement the new concept from a blank slate.

## Deactivate

To turn off Deanchor mode and return to normal, say "stop deanchor" or "normal mode", or run `/deanchor off`.
